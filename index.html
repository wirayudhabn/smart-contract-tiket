◆ PHASE 02 · CIRCULATION

TRANSFER OWNERSHIP

02

On-Chain Transfer Mechanism

▸ Transfer is called directly from the smart contract via the transferTicket() function.
▸ Only the current owner of the ticket has the right to perform the transfer (validated by the contract).
▸ Each transfer is permanently recorded as an on-chain event that can be audited by anyone.

⚐ Security Note

The previous owner automatically loses access after a successful transfer. This system prevents Double-Spending — the same ticket cannot be used more than once or by more than one party at the same time.

ROLE-BASED ACCESS CONTROL
03

Gatekeeper Verification Protocol

▸ The validateTicket() function can only be called by the admin/promoter.
▸ Validated tickets will be marked as isUsed = true and cannot be reused.
▸ The validation process emits a TicketValidated event, which is permanently recorded on the blockchain.

🔐 Role-Based Access Control

Access control is implemented on two levels: the UI layer (overlay + disabled state) and the smart contract layer (contract.admin() check), providing multi-layered protection against unauthorized access.
